MODELOS DO BANCO DE DADOS (Prisma):

1. ActiveLead            - Leads em conversação ativa
   - phone (único), name, email, company
   - score (0-100), intentClassified, status
   - firstMessageAt, lastMessageAt, messageCount
   - chatwootContactId, chatwootConvId
   - Índices: phone, status, score, lastMessageAt

2. ColdLead             - Leads arquivados
   - originalLeadId, phone, name, email, company
   - lastScore, lastStatus, conversationData
   - archivedAt, freezeReason, reactivationCount

3. Message              - Histórico de mensagens
   - leadId, content, type (incoming/outgoing)
   - isAiGenerated, scoreChange, intentDetected
   - chatwootMessageId, syncStatus

4. SystemLabel          - Etiquetas do Chatwoot
   - title (único), emoji, color
   - chatwootLabelId (único), category
   - syncedAt, lastVerifiedAt

5. Note                 - Notas privadas
   - leadId, content, type (private/sales_note/system/summary)
   - chatwootNoteId, postedToChat

6. JobLog               - Log de execução de jobs
   - jobType, status (PENDING/RUNNING/SUCCESS/FAILED/PARTIAL)
   - leadsProcessed, leadsSkipped, error
   - startedAt, completedAt, duration

7. ConversationMetric   - Métricas de conversação
   - leadId, firstResponseTime, avgResponseTime
   - messageCount, userMessageCount, aiMessageCount
   - converted, conversionTime

8. SystemHealth         - Status das integrações
   - chatwootStatus, openaiStatus, uazapiStatus
   - dbConnections, dbLatency, queueLength
   - lastChecked

ENDPOINTS DA API:

GET /health                           - Health check
GET /api/leads                        - Lista todos (limit 50)
GET /api/leads/:phone                 - Detalhes + mensagens
GET /api/leads/hot                    - Score > 80

POST /webhooks/uazapi/message         - Webhook WhatsApp
POST /webhooks/chatwoot/message-created - Webhook Chatwoot

JOBS AUTOMÁTICOS (via Agenda.js + MongoDB):

1. follow-up-24h
   - Frequência: A cada hora
   - Função: Envia follow-up para leads sem resposta há 24h
   - Limita: 50 leads por execução
   - Log: JobLog com status

2. cold-storage-7d
   - Frequência: 2x por dia (12 horas)
   - Função: Arquiva leads inativos por 7 dias com score < 50
   - Destino: Tabela ColdLead
   - Preserva: Histórico completo para reativação

COMO INICIAR:

1. Entrar no diretório:
   cd /root/home/agente\ de\ i.a/sdr-agent

2. Ver START_HERE.txt:
   cat START_HERE.txt

3. Executar setup automático:
   bash QUICK_START.sh
   # Escolher opção 1

4. Ou setup manual:
   cp .env.example .env  # Editar com credenciais
   npm install
   npm run build
   docker-compose up -d
   npm run dev

5. Testar:
   curl http://localhost:3000/health
